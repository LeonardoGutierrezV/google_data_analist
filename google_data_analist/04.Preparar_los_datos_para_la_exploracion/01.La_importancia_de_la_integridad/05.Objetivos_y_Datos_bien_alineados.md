# Objetivos y Datos bien alineados

Cuando los datos están bien alineados con los objetivos de la empresa, se puede obtener información muy valiosa y llegar a conclusiones precisas. COMO ANALISTA DE DATOS, la alineación es algo que tendrá que juzgar. Una buena alineación significa que los datos son relevantes y pueden ayudarle a resolver un problema empresarial o a determinar un curso de acción para alcanzar un objetivo empresarial determinado. 

En esta lectura, revisará los objetivos empresariales asociados a tres escenarios. Explorará cómo unos datos limpios y unos objetivos empresariales bien alineados pueden ayudarle a llegar a conclusiones precisas. Además, aprenderá cómo las nuevas variables descubiertas durante el análisis de datos pueden hacer que establezca limitaciones de datos para que pueda mantener los datos alineados con un objetivo empresarial.

Limpieza de datos
Los datos de asistencia al seminario web parecen ajustarse al objetivo empresarial. Pero el Analista de datos y el Gerente del programa deciden que es necesario limpiar los datos antes del análisis. Creen que la limpieza de datos es necesaria porque:

El nombre de la empresa no era un campo obligatorio. Si el nombre de la empresa está en blanco, podría encontrarse a partir de la dirección de correo electrónico. Por ejemplo, si la dirección de correo electrónico es username@google.com, el campo de la empresa podría rellenarse con Google para el análisis de datos. Este paso de limpieza de datos supone que las personas con direcciones de correo electrónico asignadas por la empresa asistieron a un seminario web por motivos profesionales.

Los asistentes podían introducir cualquier nombre. Dado que se está analizando la asistencia a una serie de seminarios web, es necesario validar los nombres con direcciones de correo electrónico únicas. Por ejemplo, si Joe Cox asistiera a dos seminarios pero se registrara como Joe Cox en uno y como Joseph Cox en el otro, se contaría como dos personas diferentes. Para evitarlo, es necesario comprobar su dirección de correo electrónico única para determinar que se trata de la misma persona. Tras la validación, Joseph Cox podría cambiarse por Joe Cox para coincidir con la otra instancia.

Alineación con el objetivo empresarial + variables recién descubiertas + restricciones = conclusiones precisas
Objetivo empresarial
Una empresa de clases particulares extraescolares, A+ Education, quiere saber si hay un número mínimo de horas de clase necesarias para que los alumnos mejoren al menos un 10% sus calificaciones. 

Una imagen de una persona escribiendo en un bloc y una imagen de un medidor de calibre
El analista de datos considera que existe una buena concordancia entre los datos disponibles y el objetivo empresarial porque:

Los estudiantes entran y salen del sistema para cada sesión de tutoría y se registra el número de horas

Los resultados de las evaluaciones se registran regularmente.

Limitaciones de datos para nuevas variables
Tras examinar los datos, el Analista de datos descubre que hay otras variables a tener en cuenta. Algunos alumnos tenían sesiones semanales constantes, mientras que otros tenían sesiones programadas de forma más aleatoria, aunque su número total de horas de tutoría fuera el mismo. Los datos no se alinean tan bien con el objetivo empresarial original como se pensó en un principio, por lo que el analista añade una restricción de datos para centrarse sólo en los alumnos con sesiones semanales constantes. Esta modificación ayuda a obtener una imagen más precisa sobre el tiempo de matriculación necesario para lograr una mejora del 10% en las puntuaciones de las evaluaciones.

Puntos clave
Esperamos que estos ejemplos le den una idea de lo que debe buscar para saber si sus datos se alinean con su objetivo de negocio.

Cuando hay datos limpios y una buena alineación, puede obtener información precisa y llegar a conclusiones que los datos respaldan.

Si la alineación es buena, pero los datos necesitan ser depurados, depure los datos antes de realizar el análisis.

Si los datos sólo se ajustan parcialmente a un objetivo, piense en cómo podría modificar el objetivo, o utilice limitaciones de datos para asegurarse de que el subconjunto de datos se ajusta mejor al objetivo empresarial.
