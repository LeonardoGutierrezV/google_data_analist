# Equilibrio entre seguridad y análisis

**Seguridad de los datos** significa proteger los datos de accesos no autorizados o de la corrupción estableciendo medidas de seguridad. Por lo general, el objetivo de la seguridad de los datos es impedir que usuarios no autorizados accedan a los datos sensibles o los visualicen. Los Analistas de datos tienen que encontrar la manera de equilibrar la Seguridad de los datos con sus necesidades reales de análisis. Esto puede ser complicado: queremos mantener nuestros Datos seguros y protegidos, pero también queremos utilizarlos lo antes posible para poder hacer observaciones significativas y oportunas.

Para ello, las empresas tienen que encontrar la forma de equilibrar sus medidas de Seguridad de los datos con sus necesidades de accesibilidad. 

Por suerte, existen algunas medidas de Seguridad que pueden ayudar a las empresas a conseguirlo. Las dos de las que hablaremos aquí son la encriptación y la tokenización.

**La encriptación** utiliza un algoritmo único para alterar los Datos y hacerlos inutilizables para los usuarios y las aplicaciones que no conocen el algoritmo. Este algoritmo se guarda como una "clave" que puede utilizarse para invertir la encriptación; de modo que si usted tiene la clave, puede seguir utilizando los datos en su forma original.

**La Tokenización** sustituye los elementos de datos que desea proteger por datos generados aleatoriamente denominados "token"." Los Datos originales se almacenan en una ubicación separada y se asignan a los tokens. Para acceder a los datos originales completos, el usuario o la aplicación necesita tener permiso para utilizar los datos tokenizados y el mapeo de tokens. Esto significa que incluso si los Datos tokenizados son pirateados, los datos originales siguen estando a salvo y seguros en una ubicación separada.

**La encriptación** y **la tokenización** son sólo algunas de las opciones de Seguridad de los datos que existen. Hay muchas otras, como el uso de dispositivos de autenticación para la tecnología de IA.

Como Analista de datos junior, probablemente no será responsable de crear estos sistemas. Muchas empresas tienen equipos enteros dedicados a la seguridad de los Datos o contratan a terceras empresas especializadas en la seguridad de los Datos para crear estos sistemas. Pero es importante saber que todas las empresas tienen la responsabilidad de mantener sus Datos seguros, y comprender algunos de los posibles sistemas que podría utilizar su futuro empleador. 

Sin embargo, una cosa que puede hacer absolutamente para ayudar a conseguir el RIGHT EQUILIBRIO es utilizar las mejores prácticas de control de versiones. El Control de versión permite a todos los colaboradores de un archivo realizar un seguimiento de los cambios a lo largo del tiempo. Puede saber quién hizo qué cambios en un archivo, cuándo se hicieron y por qué. 

He aquí un ejemplo sencillo: Tal vez esté trabajando en un proyecto con un equipo de otras personas. Todos estáis colaborando en el mismo conjunto de archivos, pero cada persona es responsable de una parte diferente del proyecto. Sin el control de versiones, sería muy difícil hacer un seguimiento de quién hizo qué cambios en los archivos y cuándo. Esto daría lugar a confusiones y, lo que es peor, ¡a que la gente sobrescribiera accidentalmente el trabajo de los demás! El Control de versión es esencial para los profesionales del Análisis de datos porque permite a los usuarios colaborar eficazmente con otros y experimentar con nuevas ideas sin miedo a perder su trabajo.
