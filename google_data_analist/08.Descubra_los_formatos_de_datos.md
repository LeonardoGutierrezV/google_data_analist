Sabemos que podemos comparar diferentes películas y géneros cinematográficos. Resulta que puede hacer lo mismo con los datos y los formatos de datos. Utilicemos nuestra hoja de cálculo de datos cinematográficos para entender cómo funciona. 
Empezaremos con datos cuantitativos y cualitativos. Si revisamos la columna 
A, encontraremos los títulos de las películas. Se trata de datos cualitativos porque no se pueden contar , medir ni expresar fácilmente con números. Datos cualitativos suelen aparecer como nombre, categoría o descripción. En nuestra hoja de cálculo, los títulos de las películas y los miembros del reparto son datos cualitativos. 
El siguiente paso son los datos cuantitativos, que pueden medirse o contarse y luego expresarse como un número. Se trata de datos con una cantidad, cantidad o rango determinados. 
En nuestra hoja de cálculo, las dos últimas columnas muestran el presupuesto y los ingresos de taquilla de las películas. Los datos de estas columnas se muestran en dólares, que se pueden contar, por lo que sabemos que los datos son cuantitativos. 
Podemos profundizar aún más en los datos cuantitativos y dividirlos en datos discretos o continuos. 
Veamos primero los **datos discretos**. Se trata de datos que se cuentan y tienen un número limitado de valores. Volviendo a nuestra hoja de cálculo, encontraremos el presupuesto y los ingresos de taquilla de cada película en las columnas M y N. Ambos son ejemplos de datos discretos que se pueden contar y tienen un número limitado de valores. Por ejemplo, la cantidad de dinero que gana una película solo se puede representar con exactamente dos dígitos después del decimal para representar los centavos. No puede haber nada entre uno y dos centavos. 
**Datos continuos** se pueden medir con un temporizador y su valor se puede mostrar como un decimal con varios lugares. Imaginemos una película sobre analistas de datos en la que definitivamente voy a protagonizar algún día. Podrías expresar el tiempo de ejecución de la película en 110,0356 minutos. Incluso puedes añadir datos fraccionarios después de la coma decimal si lo necesitas. También hay datos nominales y ordinales. 
**Datos nominales** son un tipo de datos cualitativos que se clasifican sin un orden establecido. En otras palabras, estos datos no tienen una secuencia. He aquí un ejemplo rápido. Supongamos que estás recopilando datos sobre películas. Le preguntas a la gente si han visto una película determinada. Sus respuestas se presentarían en forma de datos nominales. Podrían responder «Sí», «No» o «No estoy seguro». Estas opciones no tienen un orden en particular. 
**Datos ordinales**, por otro lado, son un tipo de datos cualitativos con un orden o escala establecidos. Si le pides a un grupo de personas que clasifiquen una película del 1 al 5, algunos podrían clasificarla con un 2, otros con un 4, etc. Estas clasificaciones están ordenadas según cuánto le gustó la película a cada persona. 
Ahora hablemos de los **datos internos**, que son los datos que se encuentran dentro de los propios sistemas de una empresa. Por ejemplo, si un estudio cinematográfico hubiera compilado todos los datos de la hoja de cálculo utilizando únicamente sus propios métodos de recopilación, entonces serían sus datos internos. Lo mejor de los datos internos es que, por lo general, son más fiables y fáciles de recopilar, pero en esta hoja de cálculo es más probable que el estudio cinematográfico haya tenido que utilizar datos de otros estudios y fuentes o compartidos por ellos, ya que incluye películas que ellos no hicieron. Eso significa que recopilarían datos externos. 
**Datos externos** son, lo has adivinado, datos que viven y se generan fuera de una organización. Datos externos se vuelven particularmente valiosos cuando el análisis depende de tantas fuentes como sea posible. Lo bueno de estos datos es que están estructurados. 
**Datos estructurados** son datos que se organizan en un formato determinado , como filas y columnas. Las hojas de cálculo y las bases de datos relacionales son dos ejemplos de software que pueden almacenar datos de forma estructurada. Tal vez recuerdes nuestra exploración anterior del pensamiento estructurado, que te ayuda a agregar un framework a un problema para que puedas resolverlo de manera organizada y lógica. Puede pensar en los datos estructurados de la misma manera. Tener un framework para los datos hace que los datos se puedan buscar fácilmente y estén más preparados para el análisis. 
Como analista de datos, trabajará con una gran cantidad de datos estructurados, que normalmente tendrán la forma de una tabla, hoja de cálculo o base de datos relacional, pero a veces encontrará datos no estructurados. 
**Datos no eeestructurados** se trata de datos que no están organizados de una manera fácilmente identificable. Los archivos de audio y vídeo son ejemplos de datos no estructurados porque no hay una forma clara de identificar u organizar su contenido. Datos no estructurados pueden tener una estructura interna, pero los datos no caben perfectamente en filas y columnas como los datos estructurados.
