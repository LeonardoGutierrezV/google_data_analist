Ha aprendido cómo la clasificación y el filtrado de datos en las hojas de cálculo ayudan a los analistas de datos a personalizar la información. La personalización de los datos hace que sean más significativos y fáciles de entender, analizar y visualizar. 

También descubriste que algunas hojas de cálculo pueden ser extremadamente largas y complejas. Saber cómo concentrarse en los datos exactos que necesita y dejar de lado el resto le ayuda a centrarse en el análisis. Esto también es válido para las bases de datos. A veces, un conjunto de datos es demasiado grande para descargarlo o no cabe en una hoja de cálculo, por lo que un analista de datos utilizará SQL para crear una consulta a fin de ver los datos específicos que desea del conjunto más grande. Hemos aprendido que una base de datos es una colección de datos almacenados en un sistema de computadora y que SQL significa Lenguaje de Consulta Estructurada. 

Los analistas de datos utilizan lenguajes de consulta para comunicarse con la base de datos. En un vídeo anterior, también aprendió que una base de datos relacional contiene una serie de tablas que se pueden conectar a relaciones extranjeras. Estas relaciones se representan mediante claves principales y externas. Los analistas de datos escriben consultas para obtener datos de estas tablas. Vamos a ver cómo funciona esto. 

Empezaremos con nuestro Visor de Tablas. Aquí podemos ver qué conjuntos de datos públicos están disponibles. Revisaremos los datos antes de empezar a usarlos para hacernos una idea de qué se trata y asegurarnos de que están limpios. Algunos visores de tablas permiten previsualizar algunas filas incluso antes de escribir una consulta. Esto es útil si quieres echar un vistazo rápido para asegurarte de que el conjunto de datos será el adecuado para tu proyecto. Para mostrarle cómo funciona esto, veamos un conjunto de datos de muestra. Esta muestra la cantidad de luz solar que llega a los tejados en un año. 

Esto sería muy útil para un analista de datos que trabaje en un proyecto de energía solar, por ejemplo. Empezaremos por obtener una vista previa del conjunto de datos. Luego, seleccionaremos un subconjunto de estos datos, donde encontraremos regiones, estados, luz solar anual y más. Ahora, para ver todo el conjunto de datos, escribamos una consulta. El primer paso es averiguar el nombre completo y correcto del conjunto de datos. Para ello, seleccione el conjunto de datos, el potencial solar por código postal y seleccione la tabla de consulta. El nombre del conjunto de datos se muestra dentro de las dos comillas invertidas. 

Esto nos ayuda a leer la consulta con mayor facilidad. En este caso, también podemos eliminar las comillas invertidas y nuestra consulta seguirá ejecutándose. Las palabras que aparecen antes del punto representan el nombre de la base de datos y las palabras que aparecen después del punto representan el nombre de la tabla. Vamos a seleccionar y copiar el nombre del conjunto de datos ahora porque lo necesitaremos en un segundo. 

Ahora haremos clic en el signo más para crear una nueva consulta. La mayoría de las consultas comienzan con la palabra seleccionar y luego agregamos un espacio. Como queremos ver todo el conjunto de datos, pondremos un asterisco a continuación. El asterisco dice que queremos incluir todas las columnas. Este es un excelente acceso directo porque sin él, tendríamos que escribir todos los nombres de campo. 

A continuación, presionaremos «Return» y escribiremos FROM. FROM hace exactamente lo que parece, indica de dónde provienen los datos. Después de eso, agregaremos otro espacio. Ahora pegamos el nombre del conjunto de datos que copiamos anteriormente y finalmente ejecutamos la consulta. Ahora podemos inspeccionar cuidadosamente el conjunto de datos antes de empezar a trabajar con él. Una cosa importante a tener en cuenta es que las consultas SQL se pueden escribir de muchas maneras diferentes, pero aun así proporcionan los mismos resultados. 

Por ejemplo, podríamos haber escrito esta consulta como una larga línea de instrucciones como esta y seguiríamos obteniendo los mismos resultados. Las líneas y los espacios adicionales no afectan al resultado de la consulta , pero mantienen la consulta organizada y es más fácil de leer para usted y para los demás. Ahora, si el proyecto no requiere todos estos campos, podemos usar SQL para ver un fragmento o fragmentos de datos en particular. Para ello, especificamos un nombre de columna determinado en la consulta. Por ejemplo, tal vez solo queramos ver los datos de Pensilvania. Empezaríamos nuestra consulta de la misma manera que acabamos de enterarnos. SELECT, espacio y añada un asterisco. 

Luego, DESDE nuestra base de datos de potencial solar , pero esta vez agregaremos dónde. Agregue un espacio y un nombre de subrayado de estado , el nombre de la columna. Ahora, como solo queremos ver los datos de Pensilvania, agregamos un signo igual y la palabra Pensilvania entre comillas simples. En SQL, las comillas simples indican el principio y el final de una cadena. Por último, ejecutamos la consulta. Ahora podemos revisar los datos sobre el potencial solar solo para Pensilvania.
