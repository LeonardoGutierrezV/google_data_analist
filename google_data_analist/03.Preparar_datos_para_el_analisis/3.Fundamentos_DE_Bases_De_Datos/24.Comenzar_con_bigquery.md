BigQuery es un almacén de datos en Google Cloud Platform que se utiliza para consultar y filtrar grandes conjuntos de datos, agregar resultados y realizar operaciones complejas. A lo largo de este programa, vas a utilizar BigQuery para practicar tus habilidades de SQL y recopilar, preparar y analizar datos. En este punto, usted ha configurado su propia cuenta. Ahora, explora algunos de los elementos importantes del espacio de trabajo SQL. Esto lo preparará para las próximas actividades en las que utilizará BigQuery. Tenga en cuenta que BigQuery actualiza su interfaz con frecuencia, por lo que su consola puede ser ligeramente diferente de lo que se describe en esta lectura. No pasa nada; ¡utiliza tus habilidades de resolución de problemas para encontrar lo que necesitas!

## Iniciar sesión en BigQuery

Cuando inicie sesión en BigQuery a través de la Página de destino, se abrirá automáticamente el espacio de su proyecto. Se trata de una visión general de alto nivel de su proyecto, incluida la información del proyecto y los recursos que se están utilizando actualmente. FROM aquí, usted puede comprobar su actividad reciente.

![Imagen01](/google_data_analist/03.Preparar_datos_para_el_analisis/assets/032401.png)

Navegue hasta el BigQuery Studio de su proyecto seleccionando BigQuery en el menú de navegación y BigQuery Studio en el menú desplegable.

![Imagen02](/google_data_analist/03.Preparar_datos_para_el_analisis/assets/032402.png)

##  Componentes de BigQuery Studio
Una vez que haya navegado a BigQuery desde el espacio del proyecto, la mayoría de los componentes principales de la consola BigQuery estarán presentes: el panel de navegación , el panel del explorador y el espacio de trabajo SQL.

![Imagen03](/google_data_analist/03.Preparar_datos_para_el_analisis/assets/032403.png)

## Panel de navegación
En la página de la consola, encuentra el panel de Navegación . Así es como se navega desde el espacio del proyecto a la herramienta BigQuery. Este menú también contiene una lista de otras herramientas de datos de Google Cloud Project (GCP). Durante este programa, te centrarás en BigQuery, pero es útil comprender que GCP cuenta con una colección de herramientas conectadas que los profesionales de datos utilizan a diario.

## El panel Explorador

El panel Explorador enumera sus proyectos actuales y los proyectos destacados que haya añadido a su consola. También es donde encontrará el botón + AÑADIR, que puede utilizar para añadir conjuntos de datos.

![Imagen04](/google_data_analist/03.Preparar_datos_para_el_analisis/assets/032404.png)

Este botón abre el cuadro de diálogo Añadir que le permite abrir o importar diversos conjuntos de datos.

![Imagen05](/google_data_analist/03.Preparar_datos_para_el_analisis/assets/032405.png)

## Añadir conjuntos de datos públicos

BigQuery ofrece diversos conjuntos de datos públicos del Programa de conjuntos de datos públicos de Google Cloud. Desplácese por el cuadro de diálogo Añadir hasta la opción Conjuntos de datos públicos.

![Imagen06](/google_data_analist/03.Preparar_datos_para_el_analisis/assets/032406.png)

Seleccione Conjuntos de datos públicos. Accederá al mercado de conjuntos de datos públicos, donde podrá buscar y seleccionar conjuntos de datos públicos para añadirlos a su consola BigQuery. Por ejemplo, busque el conjunto de datos "noaa lightning" en la barra de búsqueda del Mercado. Cuando busque este conjunto de datos, encontrará los datos sobre rayos nube-tierra de la **NOAA**.

![Imagen07](/google_data_analist/03.Preparar_datos_para_el_analisis/assets/032407.png)

Seleccione el conjunto de datos para leer su descripción. Seleccione Ver conjunto de datos para crear una ficha con la información del conjunto de datos dentro del espacio de trabajo de SQL.

![Imagen08](/google_data_analist/03.Preparar_datos_para_el_analisis/assets/032408.png)

## Iniciar y examinar conjuntos de datos públicos

Ha añadido el conjunto de datos público noaa_lightning a su espacio de trabajo BigQuery, por lo que el panel Explorador muestra el conjunto de datos noaa_lightning, junto con la lista de otros conjuntos de datos públicos. Estos conjuntos de datos están anidados en bigquery-public-data. Inicie bigquery-public-data navegando hasta la parte superior del panel Explorador y seleccionando la estrella junto a bigquery-public-data.

![Imagen09](/google_data_analist/03.Preparar_datos_para_el_analisis/assets/032409.png)

Si selecciona bigquery-public-data podrá buscar y añadir conjuntos de datos públicos desplazándose por el panel del Explorador o buscándolos en la barra de búsqueda del Explorador .

Por ejemplo, es posible que desee seleccionar otro conjunto de datos públicos. Si selecciona el segundo conjunto de datos, "austin_311", se ampliará para mostrar la tabla almacenada en él, "311_service_requests"

![Imagen10](/google_data_analist/03.Preparar_datos_para_el_analisis/assets/032410.png)

Al seleccionar una tabla, su información se muestra en el área de trabajo de SQL. Seleccione la tabla 311_service_requests para examinar varias pestañas que la describen, entre las que se incluyen:

* Esquema, que muestra los nombres de las columnas del conjunto de datos

* Detalles, que contiene metadatos adicionales, como la fecha de creación del conjunto de datos

* Vista previa, que muestra las primeras filas del conjunto de datos

![Imagen11](/google_data_analist/03.Preparar_datos_para_el_analisis/assets/032411.png)

Además, puede seleccionar el botón Consulta de la barra de menús del Área de trabajo SQL para consultar esta tabla.

## El Espacio de Trabajo SQL

El último panel de menú en su consola es el Espacio de Trabajo SQL. Aquí es donde escribirá y ejecutará las consultas en BigQuery.

![Imagen12](/google_data_analist/03.Preparar_datos_para_el_analisis/assets/032412.png)

El espacio de trabajo SQL también le da acceso a su historial personal y de proyectos, que almacena un registro A de las consultas que ha ejecutado. Esto puede ser útil si desea volver a una consulta para ejecutarla de nuevo o utilizar parte de ella en otra consulta.

## Cargue sus datos

Además de ofrecer acceso a conjuntos de datos públicos, BigQuery también te permite cargar tus propios datos directamente en tu espacio de trabajo. Acceda a esta función abriendo de nuevo el menú + AÑADIR o haciendo clic en los tres puntos verticales junto al nombre de su proyecto en el panel del Explorador. Esto le dará la opción de crear su propio conjunto de datos y cargar sus propias tablas. Tendrá la oportunidad de cargar sus propios datos en una próxima actividad para practicar el uso de esta función.

## Gemini

En BigQuery también encontrarás potentes herramientas de IA que te ayudarán con tu código, como Gemini Cloud Assist y la herramienta de generación de SQL. Estas herramientas funcionan como asistentes que pueden ayudar a analizar el código y encontrar errores, sugerir modificaciones y también interactuar de forma conversacional para responder a muchas otras preguntas, ya sean técnicas o más conceptuales.

Para activar Gemini Cloud Assist, busque el icono Gemini en la esquina superior derecha de la consola BigQuery para activar un panel Gemini Cloud Assist. Aquí, puede consultar Gemini utilizando lenguaje natural.

Para utilizar la herramienta de generación SQL dentro del propio editor SQL, localice el icono de la varita mágica en la esquina superior izquierda de la ventana de consulta SQL. Al hacer clic aquí se abrirá una ventana en la que podrá pedir a Gemini que genere consultas SQL para sus datos.

Aunque funciones como la generación de SQL impulsada por IA, la finalización de código y el conocimiento de los datos dentro de BigQuery pueden acelerar su flujo de trabajo, recuerde que esta tecnología aún está evolucionando. Es posible que las sugerencias, el código o las explicaciones proporcionadas no siempre sean perfectamente precisas, óptimas o seguras. Revise y valide siempre cualquier resultado generado por IA antes de ejecutar consultas o confiar en la información. Trate la ayuda de la IA como un copiloto útil, pero mantenga la supervisión y la responsabilidad de sus consultas y análisis finales.

## Puntos clave

El espacio de trabajo SQL de BigQuery le permite buscar conjuntos de datos públicos, ejecutar consultas SQL e incluso cargar sus propios datos para analizarlos. Tanto si trabaja con conjuntos de datos públicos, como si ejecuta consultas SQL o carga sus propios datos, el espacio de trabajo SQL de BigQuery ofrece una serie de funciones que le permitirán realizar todo tipo de tareas de análisis de datos. A lo largo de este programa, utilizará BigQuery para practicar sus habilidades SQL, por lo que estar familiarizado con los principales componentes de su consola BigQuery le ayudará a navegar por ella con eficacia en el futuro
