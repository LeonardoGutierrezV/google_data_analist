Las Bases de datos permiten a los analistas manipular, almacenar y procesar datos. Esto les ayuda a buscar entre los Datos de forma mucho más eficaz para obtener las mejores estadísticas.

## Bases de datos relacionales

Una base de datos relacional es una base de datos que contiene una serie de tablas que pueden conectarse para formar relaciones. Básicamente, permiten a los analistas de datos organizar y vincular datos basándose en lo que los datos tienen en común. 

En una tabla no relacional, encontrará agrupadas todas las posibles variables que le interese analizar. Esto puede dificultar mucho la clasificación. Ésta es una de las razones por las que las bases de datos relacionales son tan comunes en el análisis de datos: simplifican muchos procesos de análisis y facilitan la búsqueda y el uso de los datos en toda una base de datos.

La normalización es un proceso de organización de los datos en una base de datos relacional. Por ejemplo, creando tablas y estableciendo relaciones entre esas tablas. Se aplica para eliminar la redundancia de los datos, aumentar su integridad y reducir su complejidad en una base de datos.

## La clave de las bases de datos relacionales

Las tablas de una base de datos relacional están conectadas por los campos que tienen en común. Puede que recuerde haber aprendido antes sobre claves primarias y foráneas. Como repaso rápido, una clave primaria es un identificador que hace referencia a una columna en la que cada valor es único. En otras palabras, es una columna de una tabla que se utiliza para identificar de forma única cada registro dentro de esa tabla. El valor asignado a la clave primaria en una fila concreta debe ser único dentro de toda la tabla. Por ejemplo, si customer_id es la clave primaria de la tabla de clientes, nunca habrá dos clientes que tengan el mismo customer_id.

Por el contrario, una clave foránea es un campo dentro de una tabla que es una clave primaria en otra tabla. Una tabla sólo puede tener una clave primaria, pero puede tener varias claves foráneas. Estas claves son las que crean las relaciones entre las tablas de una base de datos relacional, lo que ayuda a organizar y conectar los datos de varias tablas de la base de datos. 

Algunas tablas no requieren una clave primaria. Por ejemplo, una tabla de Ingresos puede tener múltiples claves foráneas y no tener una clave primaria. Una clave primaria también puede construirse utilizando varias columnas de una tabla. Este tipo de clave primaria se denomina clave compuesta. Por ejemplo, si customer_id y location_id son dos columnas de una clave compuesta para una tabla de clientes, los valores asignados a esos campos en cualquier fila deben ser únicos dentro de toda la tabla.

## ¿SQL? Está hablando mi idioma

Como ha estado aprendiendo, el Lenguaje de Consulta Estructurada (SQL) es un tipo de lenguaje de consulta que permite a los analistas de datos comunicarse con una base de datos. Así, un analista de datos utilizará SQL para crear una consulta que le permita ver los datos específicos que desea dentro de un conjunto de datos más amplio. En una base de datos relacional, los analistas de datos pueden escribir consultas para obtener datos de las tablas relacionadas. SQL es una poderosa herramienta para trabajar con bases de datos.
